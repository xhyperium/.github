# 组织语言政策（强制中文）

> **效力**：xhyperium 组织全局 **P0**（不可削弱；项目只可加严）  
> **版本**：1.0.0  
> **SSOT**：`rulesets/language.md`  
> **分发**：`scripts/setup-global-rules.sh` → `~/.claude/rules/language.md`

---

## 1. 总则（P0）

**凡由人类或 Agent 撰写、面向人类阅读的自然语言，一律使用简体中文（UTF-8）。**

包括但不限于：

| 类别 | 要求 |
|------|------|
| 组织/项目文档 | README、Profile、设计/规格/计划、ADR、CHANGELOG 叙述 |
| 代码旁白 | `//` `///` `//!`、模块说明、示例说明文字 |
| 用户可见文案 | 错误 `Display`、CLI 帮助、业务提示、告警正文 |
| Agent 产出 | 对话回复、审查意见、handoff、PR/Issue 描述与评论 |
| 提交说明 | Conventional Commits：`type(scope): 中文说明` |
| CI / 脚本说明 | workflow 注释、脚本头注释、失败时给人看的错误信息 |

**禁止**：

- 无书面豁免的大段英文文档/英文 Agent 汇报（专有名词除外，见 §2）
- 用英文 STE 替代中文作为默认交付语言
- 非 UTF-8 编码或中文二次错误转码

---

## 2. 允许保留英文的范围（白名单）

以下**不是**「改用英文写作」，而是技术载体，必须保留英文惯例：

| 允许 | 示例 |
|------|------|
| 标识符 | 函数/类型/变量/模块名：`snake_case` / `PascalCase` |
| 包与路径 | crate 名、仓库名、`uses:` 路径、URL |
| 协议与工具专有名 | `HTTP`、`JSON`、`tokio`、`clippy`、`GitHub Actions` |
| 机器格式 | YAML/JSON **键名**、日志 **field key**、env 名、CLI flag |
| 命令与代码块 | shell / Rust 源码本身 |
| 引用外链标题 | 第三方文档原文标题可保留，**本组织摘要须中文** |
| 既有稳定 wire 字符串 | 对外已发布的 error code 字符串（变更视为 breaking） |

技术术语写法：**中文叙述 + 必要英文本体**，例如「可复用 CI（reusable workflow）」「工作区依赖（`workspace.dependencies`）」。

---

## 3. Agent 强制行为（P0）

1. **默认语言中文**：对用户的全部说明、分析、审查、总结用中文。  
2. **禁止无故切英文**：用户未要求英文全文时，不得用英文长文回复。  
3. **代码可英文、解释必须中文**：贴代码后用中文说明意图与影响。  
4. **PR / commit**：标题与正文叙述中文（允许保留 issue 号、check 名等英文 token）。  
5. **与本文件冲突时**：以本文件为准（宪法 C-5：约束先于自由）。

---

## 4. 文档与仓内文件（P0）

| 路径类型 | 语言 |
|----------|------|
| `rulesets/**/*.md` | 中文（本政策下新建/修订必须中文） |
| `profile/`、根 `README`、`workflows/*.md` | 中文 |
| 模块 `README` / `AGENTS.md` / 规格与设计 | 中文 |
| 测试名/断言消息 | 优先中文或中英混合且人话可读；禁止纯英文长句堆砌业务语义 |

历史英文段落：修订触及处**改为中文**；未触及可不整仓翻译，但**新增必须中文**。

---

## 5. 与 STE100 / 多语言产品

| 项 | 政策 |
|----|------|
| ASD-STE100 | **不作为**组织默认；不进入强制加载 |
| 对外多语言产品 UI | 产品层 i18n 另册；**源码默认文案与文档母语仍为中文**，译文目录可另建 |
| 法规强制英文的交付物 | 须 **书面豁免**（PR 注明条款、范围、期限），且母语说明仍用中文归档 |

---

## 6. 违规与审查

| 级别 | 情形 | 处置 |
|------|------|------|
| P0 | 用户可见错误/公共 API 文档/Agent 交付全文无故用英文 | Review 阻断；Agent 须重写 |
| P1 | 大段英文注释或设计文档 | 要求补中文 |
| 例外 | §2 白名单 | 允许 |

---

## 7. 与其它规则的关系

```text
language.md（本文件，语言 P0）
    ↓
constitution / rust/RULES / agent-*
```

- `rust/RULES.md` §2 语言表须与本文件一致（中文优先；无「默认英文交付」）。  
- 项目宪章**不可**把「全文英文文档」升为默认；只可对白名单类加严。
